# CASCADE PHASE A (STEP A2) --- NOTATION FREEZE (VERSIONED SYMBOL DICTIONARY)

Project: `nps-proof-pipeline`

Context: - CAS-0A is certified and frozen. - Phase A operates with
provisional/validation objects. - To prevent drift, Phase A must
**freeze notation** before implementing structural exposure/coordinates.

This step introduces a **versioned notation freeze** for the
SymbolDictionary and produces a Phase A notation snapshot doc.

------------------------------------------------------------------------

## 🎯 Objective

1.  Add a versioned freeze mechanism to the SymbolDictionary.
2.  Prevent mutation (adding/redefining symbols) after freeze.
3.  Generate a Phase A notation snapshot in documentation.
4.  Add tests ensuring freeze behaviour and snapshot generation
    invariants.

No mathematics is introduced; this is governance + coherence
infrastructure.

## 1️⃣ Add Freeze Support to SymbolDictionary

In SymbolDictionary, implement:

-   `freeze(version: str) -> None`
-   `is_frozen() -> bool`
-   `frozen_version` property/attribute (Optional\[str\] or str \| None)

Rules:

-   Before freeze: normal behaviour
-   After freeze:
    -   adding a symbol must raise
        `MathematicalInconsistency("SymbolDictionary is frozen")`
    -   redefining an existing symbol must raise the same error
-   `freeze(version)` must:
    -   raise
        `MathematicalInconsistency("SymbolDictionary already frozen")`
        if called twice
    -   store the version string (must be non-empty; empty should raise)

Use existing exception type:
`nps.core.exceptions.MathematicalInconsistency`

``` python
    kind: str

    codomain: str
    regularity: str
    locality: str
    source: str

def render_symbol_dictionary_markdown(symbols: SymbolDictionary) -> str:
    """Render a deterministic markdown snapshot of the current SymbolDictionary."""
    ...
```

Requirements for `render_symbol_dictionary_markdown`:

-   Deterministic ordering (sort by symbol name)
-   Include a header with:
    -   snapshot purpose
    -   freeze version if frozen, else "UNFROZEN"
    -   current date NOT required (avoid non-determinism)
-   Render a markdown table with columns matching SymbolSnapshotRow.
-   If SymbolDictionary stores richer metadata, map it to strings;
    otherwise use placeholders like "UNKNOWN" but be consistent.

This file must not touch the filesystem; it returns markdown text only.

## 4️⃣ Tests

Add `tests/phase_a/test_symbol_dictionary_freeze_phasea.py`:

### 4.1 Freeze prevents mutation

-   Create a SymbolDictionary (use existing construction pattern).
-   Add at least one symbol successfully.
-   Call `freeze("CAS-A.draft-0")`.
-   Assert:
    -   `is_frozen()` is True
    -   `frozen_version == "CAS-A.draft-0"`
-   Attempt to add another symbol: must raise MathematicalInconsistency
    with message containing "frozen".
-   Attempt to redefine an existing symbol: must raise too.
-   Calling `freeze(...)` again must raise "already frozen".
-   Calling `freeze("")` must raise.

### 4.2 Snapshot renderer deterministic

-   Create a SymbolDictionary with two symbols.
-   Render markdown twice.
-   Assert strings equal.

Do not assert on full markdown content except stable structure; focus on
determinism.
